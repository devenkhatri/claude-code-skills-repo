01

What This Project Does

A collection of skills that make your AI coding assistant do more — with less effort from you.

Imagine you have a super-smart assistant that can write code for you. But every time you need something specific — like turning code into a tutorial, or creating a client presentation — you have to explain the whole process from scratch.

That's exhausting. This project solves that by giving your AI assistant skills.

💡
Why This Matters for You

When you use AI to code, you give it instructions in plain English. But complex tasks need many instructions. Skills package those instructions into one command — like a macro for your AI assistant.

📚

🎯

02

Meet the Skills

Each skill has a specific job — here's what they bring to the table.

📚
Codebase-to-Course

Takes any folder of code and transforms it into a beautiful, interactive HTML course. Uses scroll-based navigation, animated diagrams, and plain-English explanations.

🎯
Codebase-to-Demo

Transforms code into a sales demo for clients. Includes before/after comparisons, architecture maps, and sequence diagrams — everything you need to pitch your work.

📊
Excalidraw Diagram

Creates diagrams from natural language descriptions. Architecture, flows, mind maps — describe what you want, get a diagram.

🛠️
Skill Builder

A meta-skill that helps you create new skills. Contains best practices, templates, and guidance for building reusable AI workflows.

🤔
Why Four Separate Skills?

Each skill does one thing really well. This follows a key software principle: separation of concerns. When you need a course, you use the course skill. When you need a diagram, you use the diagram skill. No mixing, no confusion.

What the Code Looks Like

PLAIN ENGLISH

Line 1: The skill's short name — this is what you type after the slash command.

Line 2: A description that tells the AI when to use this skill. It's like a keyword match.

Blank line: Just separating sections for readability.

Lines 5-7: The step-by-step process the AI follows. Each numbered item is a concrete action.

📝
The .md Files

Each skill is defined in a .md file (Markdown). Markdown is just plain text with simple formatting — bold, lists, headers. Easy to read, easy to edit. That's why skills are so approachable: they're just documents.

The Outside World
