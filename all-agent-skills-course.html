What This Project Does

Imagine you have a super-smart assistant, but every time you need it to do something specific, you have to re-explain the whole process from scratch. This project fixes that.

The problem it solves

AI coding assistants like Claude are brilliant generalists — but they have no persistent memory of your workflows. Every session starts fresh.

😫
Without skills:

"Please create a LinkedIn carousel. Make it premium, 8 slides, 4:5 ratio, SVG format, dark background, accent color #D94F30, content arc starting with a hook and ending with a CTA, each slide with a title and 3 bullet points, font Bricolage Grotesque…"

✨
With the carousel-creator skill:

"Create a carousel about building habits." — Done. All those instructions are already baked into the skill.

What a skill actually is

A skill is a Markdown file that lives in a folder inside your project. When you invoke it with a slash command, Claude Code reads it and follows its instructions exactly.

💡
The Key Insight

Skills are like macros for your brain. Instead of re-thinking the same process, you capture it once in a SKILL.md and invoke it with one command. The AI reads the file and delivers the output — no re-explanation needed.

The slash command is just a shortcut

A slash command is mapped in Claude Code's project settings. It's like a speed-dial button — it instantly routes you to the right SKILL.md file without you having to navigate there manually.

CODE 
# .claude/settings.json
{
  "skills": {
    "/carousel-creator":
      "carousel-creator/SKILL.md",
    "/ui-ux-auditor":
      "ui-ux-auditor/SKILL.md"
  }
}
PLAIN ENGLISH

This is the project config file that maps slash commands to their skill files.

When you type /carousel-creator, Claude Code looks up this file and finds the path to the carousel skill.

Same for /ui-ux-auditor — each command has its own dedicated instruction file.

You could add your own skills here by adding new entries.

💡
Why This Matters for Steering AI

When something goes wrong, you now know exactly where to look — the SKILL.md file for that command. You can open it, read the instructions Claude is following, and either edit them or override them in your message.

# Carousel Creator

You are an SVG-based Instagram
Carousel Creator. Generate actual
carousel slides as SVG files only.

PLAIN ENGLISH

The front matter (between the dashes) is metadata Claude uses to identify and describe the skill.

The "name" is the skill's ID — must match the folder name exactly.

The "description" tells Claude when to activate this skill automatically.

Below the dashes is the actual instruction set — written in plain English for Claude to follow.

These lines tell Claude the exact pixel dimensions for every slide. No guessing.

Tracing Data Through a Skill

Let's follow one real request end-to-end — from your words to the final SVG files on disk.

The carousel-creator in action

You say: "Create a carousel about building habits." Eight things happen in sequence — like a relay race where each runner hands the baton perfectly.

1
Command matched → SKILL.md loaded

Claude Code sees /carousel-creator and loads carousel-creator/SKILL.md into its context window before doing anything else.

2
Topic analyzed → content arc planned

The skill instructs Claude to think about the audience, create a Hook → Context → Value → CTA narrative arc, and decide on 6-8 slide titles before writing any SVG.

3
Design palette selected

No brand colors specified? The skill tells Claude to choose a professional palette appropriate to the topic — avoiding clichéd purple AI gradients.

4
SVG generated for each slide

Claude writes complete SVG code for each slide at 1080×1350px with embedded fonts, proper viewBox, and all content — one file per slide.

5
Validation checklist run

The skill includes a built-in QA step: check every SVG for unclosed tags, missing width/height/viewBox, invalid CSS. Self-corrects if found.

6
Files saved to timestamped folder

Output goes to YYYYMMDD-HHMM/slides/ — a new folder for each session, so previous carousels are never overwritten.

7
caption.md written

A ready-to-paste caption with hashtags is saved alongside the slides — so you have the full post package in one folder.

8
Summary reported back

Claude tells you what was created, where it was saved, and any design choices it made (palette, slide count, content arc used).

Why the output folder is timestamped

The folder naming convention — 20250506-1430/slides/ — is an intentional design decision in the skill. It's not random.

CODE (from SKILL.md) 
## Output Structure

Save all files to:
YYYYMMDD-HHMM/slides/

- slide-01.svg
- slide-02.svg
- ...
- caption.md

# Never overwrite existing output.
# Each run = new timestamped folder.
PLAIN ENGLISH

Every carousel run creates its own folder named with the date and time.

This means you can run the skill 10 times in a day and keep all versions — nothing gets accidentally overwritten.

It also makes it easy to find yesterday's carousel vs today's — just look at the folder timestamp.

The comment (lines starting with #) are notes for the AI, not code to execute.

🗂️
The "Version Control by Timestamp" Pattern

This is a classic technique when you can't or don't want to use proper version control for generated assets. Each run is self-contained and can be safely deleted without touching others.

06

The Clever Tricks

Skilled engineers don't just solve problems — they solve them in ways that prevent future problems. Here's what's clever about how these skills are built.

Self-healing output

The carousel-creator doesn't just generate SVG and hope for the best. It has a validation step baked in:

✅

Structural checks

Scans every SVG for unclosed tags, missing viewBox attributes, and invalid nesting — common AI mistakes.

🔄

Auto-correction

If a check fails, the skill fixes the specific issue and re-saves. No broken files reach your output folder.

⚡

Immediate recovery

Recovery happens within the same generation pass — you never see the broken intermediate state.

Reference file architecture

Complex skills like codebase-to-course don't put all instructions in one giant file. They use a references/ folder — like a binder with tabbed sections:

codebase-to-course/
SKILL.mdMain instructions — tells Claude to read reference files at the right phase
references/
design-system.mdAll CSS tokens, typography, colors — read before writing any CSS
interactive-elements.mdChat animations, flow diagrams, code blocks — read before building interactives
💡
Just-in-Time Loading

The SKILL.md tells Claude: "Read design-system.md before writing CSS, read interactive-elements.md before building animations." This is just-in-time loading — it keeps Claude's focus sharp at each phase instead of trying to hold everything in mind simultaneously.

The 4-phase build pattern

The codebase-to-demo and codebase-to-course skills both use the same disciplined 4-phase approach — like a construction project that follows a strict order:

1

Deep-read the codebase

→
2

Design the structure

→
3

→
4

The skill explicitly tells Claude: "Don't try to write all 8 modules in one pass — quality drops. Build Module 1, verify it works, then Module 2." This prevents the quality degradation that happens when AI tries to do too much at once.

Build Your Own Skill

The skill-builder skill is a meta-skill — it uses the same framework to help you create new skills for any workflow you repeat.

The 6-round Discovery Interview

When you invoke /skill-builder, Claude Code doesn't immediately ask you to describe your skill. It runs a structured interview — like a consultant onboarding a new client:

R1
Trigger & Context

What makes you reach for this skill? What do you have as input when you start?

R2
Output & Success

What does "done" look like? What file, output, or result do you need?

R3
The Process

Walk through the steps you currently follow manually — in order.

R4
Quality Standards

What makes a great output vs. a mediocre one? What are your non-negotiables?

R5
Edge Cases

What goes wrong? What inputs are tricky? What should the skill do if something fails?

R6
Examples & References

Do you have examples of great outputs? Any reference files the skill should use?

What you get

After the 6 rounds, the skill-builder produces a complete, production-ready SKILL.md file and installs it in .claude/skills/ automatically.

📋

Structured SKILL.md

Following the same proven format used by all 7 skills in this repo — YAML frontmatter, clear sections, explicit instructions.

⚡

Ready Immediately

The skill is installed and usable the moment it's generated. No extra setup, no config file editing.

♻️

Reusable Forever

Once built, your skill works identically every time — 10 or 1,000 invocations later. The process is permanently captured.

🕐
Time to Build

A typical new skill takes 20–30 minutes from idea to working implementation. The interview is thorough but fast — and the time you invest pays back on every future use of the skill.
