06

Why these formats? Why this approach?

Every design decision exists for a practical reason — not a technical preference.

🖼️
SVG for Carousels
carousel-creator output

SVGs are infinitely scalable — your slide looks pixel-perfect on a phone and a 4K display. Editable in any browser, directly importable to Canva or Figma, and usable as-is without any conversion step.

vs. HTML/CSS: HTML carousels require a browser to render at fixed viewport sizes. SVG is a single file that works everywhere, instantly, with no dependencies.
📄
Self-Contained HTML
demos, courses, slides output

One file. No server, no login, no install. Open in any browser, share via email, attach to a message, work completely offline. Your client receives something they can actually open — not a link that might break.

vs. Notion / PowerPoint: Proprietary formats lock you in. A self-contained HTML file lives forever, works anywhere, and can be white-labeled or embedded with zero friction.
🤖
Claude Code Skills
core delivery mechanism

Skills live inside your development environment — not a SaaS subscription, not a Chrome extension, not a separate tool. They run where you already work and access your local files directly without copy-paste.

vs. standalone tools: No context-switching, no API key management, no monthly fee beyond what you already pay for Claude. Everything runs in one place.
📐
Excalidraw JSON
excalidraw-diagram output

Diagrams that are fully editable after generation. Drop the .excalidraw file into excalidraw.com and every component is movable, re-labelable, and shareable — not a static PNG you can't touch.

vs. Mermaid / PlantUML: Text-based diagram formats produce diagrams you can't interact with. Excalidraw JSON produces something you can hand to a designer or client to iterate on visually.
